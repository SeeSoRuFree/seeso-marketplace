# 커리어리 의사결정 추적

프로젝트에서 내린 의사결정의 맥락과 이유를 추적합니다.

## 사용법

```
/careerly-context:decision [질문]
```

예시:
- `/careerly-context:decision 왜 카카오 로그인을 선택했지?`
- `/careerly-context:decision SSR vs CSR 결정 이유`
- `/careerly-context:decision MCP 도입 배경`

## 검색 전략

질문: $ARGUMENTS

### Step 1: 키워드 추출
질문에서 의사결정 관련 키워드 추출

### Step 2: 다중 소스 검색

**슬랙 검색**:
```
mcp__slack__conversations_search_messages(
    search_query="[키워드] 결정 OR 선택 OR 이유",
    limit=20
)
```

**로컬 문서 검색**:
```
Grep(pattern="[키워드]", glob="**/*.md")
```

### Step 3: 의사결정 정리

```markdown
## 의사결정 추적: [주제]

### 검색 결과

**출처: 슬랙 #채널 (날짜)**
- 관련 내용...

**출처: 문서명**
- 관련 내용...

### 의사결정 요약

- **결정일**: YYYY-MM-DD (추정/확인)
- **결정자**: @이름 (알 수 있는 경우)
- **맥락**: 왜 이 결정이 필요했는지
- **검토 대안**:
  1. 대안 1
  2. 대안 2
- **선택 이유**: 왜 이 옵션을 선택했는지
- **영향 범위**: auth, user-flow 등

### 추가 조사 필요

명확한 기록이 없는 경우:
- 추천 검색 채널/기간
- 관련자 문의 제안
```

## 의사결정 기록 형식

새로운 의사결정 발견 시 다음 형식으로 기록 권장:

```markdown
### 의사결정: [제목]

- **결정일**: YYYY-MM-DD
- **결정자**: @이름
- **맥락**: 왜 이 결정이 필요했는지
- **검토 대안**:
  1. 대안 1
  2. 대안 2
- **선택 이유**: 왜 이 옵션을 선택했는지
- **영향 범위**: auth, user-flow 등
- **출처**: 슬랙 #채널 또는 문서 경로
```
