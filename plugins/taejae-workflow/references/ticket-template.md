# 티켓 템플릿

## 기본 티켓 구조

### Notion 페이지 구조

```markdown
# [티켓 제목]

## 요청 개요
[요청에 대한 간단한 설명]

## 상세 내용
[상세한 요청 내용]

## 영향 범위
- 페이지: [영향 받는 페이지 목록]
- 컴포넌트: [영향 받는 컴포넌트]

## 첨부 자료
- [첨부 파일 목록]

## 작업 기록
| 날짜 | 상태 | 내용 |
|------|------|------|
| 2025-01-30 | 생성 | 티켓 생성됨 |

## 메모
[추가 메모 사항]
```

## 유형별 티켓 템플릿

### 1. 콘텐츠 교체 티켓

```markdown
# [컴포넌트명] [콘텐츠종류] 교체

## 요청 개요
[페이지명]의 [위치]에 있는 [콘텐츠]를 교체 요청

## 상세 내용
- **현재 상태**: [현재 콘텐츠 설명]
- **요청 상태**: [교체 후 콘텐츠 설명]
- **교체 대상 파일**: [파일 경로]

## 영향 범위
- 페이지: [페이지 경로]
- 컴포넌트: [컴포넌트명]
- 파일: [수정될 파일 경로]

## 첨부 자료
- 새 이미지/PDF: [파일명]
- 참고 스크린샷: [파일명]

## 작업 체크리스트
- [ ] 기존 파일 백업
- [ ] 새 파일 적용
- [ ] 로컬 확인
- [ ] 배포
- [ ] 실서버 확인

## 분류 정보
- 난이도: 쉬움
- 배포: 즉시
- 타입: 콘텐츠
```

### 2. 스타일 수정 티켓

```markdown
# [컴포넌트명] 스타일 수정

## 요청 개요
[컴포넌트]의 [스타일 요소]를 변경 요청

## 상세 내용
- **현재 상태**: [현재 스타일]
- **요청 상태**: [변경할 스타일]

### 변경 사항
| 요소 | Before | After |
|------|--------|-------|
| 색상 | #000 | #333 |
| 크기 | 16px | 18px |

## 영향 범위
- 페이지: [영향 페이지 목록]
- 컴포넌트: [컴포넌트명]
- 스타일 파일: [SCSS 파일 경로]

## 첨부 자료
- 디자인 시안: [파일명]
- 참고 이미지: [파일명]

## 작업 체크리스트
- [ ] 스타일 변경
- [ ] 반응형 확인
- [ ] 크로스 브라우저 확인
- [ ] 로컬 확인
- [ ] 배포

## 분류 정보
- 난이도: 보통
- 배포: 예정
- 타입: 프론트엔드
```

### 3. 기능 추가 티켓

```markdown
# [기능명] 추가

## 요청 개요
[페이지/컴포넌트]에 [기능] 추가 요청

## 상세 내용

### 기능 요구사항
1. [요구사항 1]
2. [요구사항 2]
3. [요구사항 3]

### 사용자 시나리오
1. 사용자가 [행동]
2. 시스템이 [반응]
3. 결과: [결과]

## 영향 범위
- 페이지: [페이지 목록]
- 컴포넌트: [컴포넌트 목록]
- 새 파일: [생성될 파일]
- 수정 파일: [수정될 파일]

## 기술 고려사항
- 의존성: [필요한 라이브러리/모듈]
- API: [필요한 API 엔드포인트]
- 데이터: [필요한 데이터 구조]

## 첨부 자료
- 기획서: [파일명]
- 디자인: [파일명]
- 참고 자료: [파일명]

## 작업 체크리스트
- [ ] 설계 검토
- [ ] 컴포넌트 구현
- [ ] 기능 구현
- [ ] 테스트
- [ ] 코드 리뷰
- [ ] 배포

## 분류 정보
- 난이도: 어려움
- 배포: 검토필요
- 타입: 복합
```

### 4. 버그 수정 티켓

```markdown
# [버그 요약] 수정

## 요청 개요
[페이지/기능]에서 [버그 현상] 발생

## 상세 내용

### 버그 상황
- **발생 위치**: [페이지/컴포넌트]
- **재현 조건**: [버그 재현 방법]
- **기대 동작**: [정상 동작]
- **실제 동작**: [버그 동작]

### 재현 단계
1. [단계 1]
2. [단계 2]
3. [버그 발생]

## 영향 범위
- 영향 페이지: [페이지 목록]
- 영향 사용자: [영향 받는 사용자 범위]

## 첨부 자료
- 버그 스크린샷: [파일명]
- 에러 로그: [로그 내용]

## 작업 체크리스트
- [ ] 원인 분석
- [ ] 수정
- [ ] 회귀 테스트
- [ ] 배포

## 분류 정보
- 난이도: [분석 후 결정]
- 배포: [긴급도에 따라]
- 타입: [영향 영역에 따라]
```

## 티켓 ID 규칙

### 형식

```
TICKET-YYYYMMDD-NNN

예시: TICKET-20250130-001
```

### 구성
- `TICKET`: 고정 접두사
- `YYYYMMDD`: 생성 날짜
- `NNN`: 당일 순번 (001부터)

## 상태 전이

```
┌─────────────────────────────────────┐
│                                     │
▼                                     │
[대기] ──→ [진행중] ──→ [완료]        │
  │           │                       │
  │           └──→ [보류] ───────────┘
  │                  │
  └──────────────────┘
```

### 상태 설명

| 상태 | 설명 | 전이 가능 |
|------|------|----------|
| 대기 | 작업 대기 중 | 진행중, 보류 |
| 진행중 | 작업 진행 중 | 완료, 보류 |
| 완료 | 작업 완료 | - |
| 보류 | 일시 중단 | 대기, 진행중 |
