# 분류 상세 규칙

## 난이도 분류 의사결정 트리

```
시작
  │
  ├─ 코드 변경 필요? ─No─→ [쉬움] (콘텐츠만)
  │
  Yes
  │
  ├─ 로직 변경 필요? ─No─→ [쉬움/보통]
  │                         │
  │                         ├─ 파일 1개? ─→ [쉬움]
  │                         └─ 파일 여러개? ─→ [보통]
  Yes
  │
  ├─ 외부 연동 필요? ─Yes─→ [어려움]
  │
  No
  │
  ├─ 새 컴포넌트 필요? ─Yes─→ [보통/어려움]
  │                           │
  │                           ├─ 템플릿 있음? ─→ [보통]
  │                           └─ 새로 설계? ─→ [어려움]
  No
  │
  └─ 기존 컴포넌트 수정? ─→ [보통]
```

---

## 난이도별 상세 기준

### 쉬움 (Easy) - 점수: 1-3점

#### 판단 기준 (OR 조건)

| 기준 | 점수 |
|------|------|
| 이미지/텍스트/PDF 단순 교체 | 1 |
| 단일 파일 수정 | 1 |
| 복사-붙여넣기 수준 | 1 |
| 기존 패턴 그대로 사용 | 1 |
| 빌드 확인만 필요 | 1 |

#### 대표 작업

```
✓ public/resources/img/*.jpg 교체
✓ public/resources/pdf/*.pdf 교체
✓ 텍스트 문구 수정
✓ 링크 URL 수정
✓ 기존 컴포넌트 props 값 변경
```

---

### 보통 (Medium) - 점수: 4-6점

#### 판단 기준 (복합 조건)

| 기준 | 점수 |
|------|------|
| 여러 파일 수정 필요 | 2 |
| CSS/SCSS 수정 필요 | 2 |
| 새 컴포넌트 추가 (템플릿 기반) | 2 |
| 조건부 렌더링 추가 | 2 |
| 라우트 추가 | 2 |
| 반응형 대응 필요 | 1 |
| 기존 컴포넌트 구조 수정 | 2 |

#### 대표 작업

```
✓ 새 페이지 추가 (기존 템플릿 활용)
✓ 기존 컴포넌트에 옵션 추가
✓ 레이아웃 변경
✓ 새 섹션 추가
✓ 네비게이션 메뉴 항목 추가
✓ 모달/팝업 추가
```

---

### 어려움 (Hard) - 점수: 7점 이상

#### 판단 기준 (복합 조건)

| 기준 | 점수 |
|------|------|
| 새 기능 설계 필요 | 3 |
| 상태 관리 추가/변경 | 3 |
| 외부 API 연동 | 3 |
| 폼 + 유효성 검증 | 3 |
| 복잡한 인터랙션 | 2 |
| 데이터 처리 로직 | 2 |
| 보안 고려 필요 | 2 |
| 성능 최적화 필요 | 2 |

#### 대표 작업

```
✓ 문의 폼 + 이메일 발송
✓ 검색 기능 구현
✓ 필터/정렬 기능
✓ 동적 콘텐츠 로딩
✓ 인증/권한 기능
✓ 파일 업로드 기능
```

## 배포 가능 분류 기준

### 즉시 (Immediate)

```
조건 (모두 충족):
□ 난이도가 '쉬움'
□ 로직 변경 없음
□ 콘텐츠 변경만 있음
□ 기존 기능에 영향 없음

또는:
□ 긴급 버그 수정 (핫픽스)
```

### 예정 (Scheduled)

```
조건 (하나라도 충족):
□ 난이도가 '보통'
□ UI/UX 변경 포함
□ 여러 티켓 함께 배포 권장
□ QA 테스트 필요
```

### 검토필요 (Requires Review)

```
조건 (하나라도 충족):
□ 난이도가 '어려움'
□ 새 기능 추가
□ 외부 연동 포함
□ 보안 관련 변경
□ 비가역적 변경 (DB 스키마 등)
□ 다른 시스템에 영향
```

## 타입 분류 규칙

### 키워드 기반 분류

#### 콘텐츠 (Content)

```regex
패턴: 이미지|사진|그림|텍스트|문구|PDF|문서|파일|링크|URL|배너|로고

점수 조건:
- 키워드 매칭: +3점
- 파일 확장자 (jpg/png/pdf): +2점
- public/resources 경로: +2점

임계값: 5점 이상 → 콘텐츠
```

#### 프론트엔드 (Frontend)

```regex
패턴: 레이아웃|디자인|스타일|색상|폰트|크기|위치|정렬|버튼|UI|UX|반응형|모바일|애니메이션

점수 조건:
- 키워드 매칭: +3점
- .scss/.css 파일 변경: +2점
- 컴포넌트 JSX 변경: +2점
- 스타일 관련 props: +1점

임계값: 5점 이상 → 프론트엔드
```

#### 백엔드 (Backend)

```regex
패턴: 폼|양식|제출|전송|저장|데이터|API|서버|이메일|메일|알림|로그인|회원

점수 조건:
- 키워드 매칭: +3점
- API 호출 추가: +3점
- 데이터 처리 로직: +2점
- 외부 서비스 연동: +2점

임계값: 5점 이상 → 백엔드
```

#### 복합 (Both)

```
조건:
- 프론트엔드 점수 >= 3 AND 백엔드 점수 >= 3
- 또는 명시적으로 '프론트 + 백엔드' 언급
```

## 분류 결과 보정

### 자동 분류 후 검토 항목

1. **콘텐츠 → 프론트엔드 승격**
   - 콘텐츠 교체 시 크기/위치 변경 필요
   - 새 이미지에 맞는 스타일 조정

2. **프론트엔드 → 복합 승격**
   - UI 변경 시 데이터 처리 필요
   - 폼 UI에 백엔드 연동 필요

3. **쉬움 → 보통 승격**
   - 단순해 보이지만 테스트 필요
   - 여러 페이지에 영향

4. **보통 → 어려움 승격**
   - 예상보다 의존성 복잡
   - 기존 코드 리팩토링 필요

## 분류 예시

### 예시 1: 배너 이미지 교체

```
요청: "메인 페이지 배너 이미지를 새 이미지로 교체해주세요"

분석:
- 키워드: 이미지, 교체, 배너 → 콘텐츠
- 파일: public/resources/img/ → +2점
- 로직 변경: 없음 → 쉬움
- 리스크: 낮음 → 즉시 배포

결과:
- 난이도: 쉬움
- 배포: 즉시
- 타입: 콘텐츠
```

### 예시 2: 문의 폼 추가

```
요청: "문의하기 페이지에 연락처 입력 폼을 추가하고 이메일로 발송해주세요"

분석:
- 키워드: 폼, 이메일, 발송 → 백엔드
- 키워드: 페이지, 추가 → 프론트엔드
- 새 기능: 폼 + 이메일 → 어려움
- 외부 연동: 이메일 서비스 → +3점
- 리스크: 중간 → 검토 필요

결과:
- 난이도: 어려움
- 배포: 검토필요
- 타입: 복합
```

### 예시 3: 새 페이지 추가

```
요청: "장학금 안내 페이지를 새로 추가해주세요"

분석:
- 키워드: 페이지, 추가 → 프론트엔드
- 작업: 새 컴포넌트 생성 → 보통
- 템플릿: 기존 안내 페이지 참고 가능 → 보통 유지
- 라우트 추가 필요 → +2점

결과:
- 난이도: 보통
- 배포: 예정
- 타입: 프론트엔드
```
