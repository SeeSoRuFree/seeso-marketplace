# taejae-deploy-notify

배포 완료 후 고객에게 통보할 메일 초안을 작성하는 스킬입니다.

## 트리거

- `/deploy-notify` - 배포 통보 메일 작성
- `/deploy-notify --template <템플릿명>` - 특정 템플릿 사용
- `/deploy-notify --preview` - 메일 미리보기만

## 개요

배포 완료 후 고객에게 발송할 이메일 초안을 자동으로 생성합니다:
- 변경사항 요약
- 확인 방법 안내
- 한국어 비즈니스 이메일 형식

**중요**: 메일은 자동 발송되지 않습니다. 생성된 초안을 확인 후 수동으로 복사하여 발송해야 합니다.

## 입력 정보

1. **변경사항 설명**: `changelog-writer` 에이전트의 출력
2. **티켓 정보**: 완료된 티켓 정보
3. **배포 정보**: 배포 일시, 환경 정보

## 메일 구성요소

### 1. 제목 (Subject)

```
[태재대학교 입학처] 웹사이트 업데이트 완료 안내 - [날짜]
```

또는 구체적인 내용 포함:
```
[태재대학교 입학처] 메인 배너 이미지 교체 완료 안내
```

### 2. 인사말 (Greeting)

```
안녕하세요, [담당자명/부서명] 담당자님.

태재대학교 입학처 웹사이트 관리를 담당하고 있는 [발신자]입니다.
```

### 3. 본문 (Body)

- 완료된 작업 요약
- 변경 내용 목록
- 확인 방법 안내

### 4. 맺음말 (Closing)

```
추가 수정이 필요하시거나 문의사항이 있으시면 언제든 연락 주시기 바랍니다.

감사합니다.

[서명]
```

## 완료된 작업

다음 항목들이 업데이트되었습니다:

1. **메인 페이지 배너 이미지 교체**
   - 메인 페이지 상단 슬라이드의 3번째 이미지가 새 이미지로 교체되었습니다.

2. **모집요강 PDF 업데이트**
   - 입학 안내 페이지의 2025학년도 모집요강 PDF가 최신 버전으로 업데이트되었습니다.

## 확인 방법

변경사항을 다음과 같이 확인하실 수 있습니다:

### 메인 페이지 배너
1. 웹사이트 메인 페이지 접속
2. 상단 배너 슬라이드에서 좌우 화살표로 3번째 이미지 확인

### 모집요강 PDF
1. 입학 안내 페이지 접속
2. "모집요강 다운로드" 버튼 클릭
3. 다운로드된 PDF 확인

> 💡 **참고**: 이전 내용이 보이는 경우, 브라우저 새로고침(Ctrl+F5 또는 Cmd+Shift+R)을 시도해주세요.

## 메일 템플릿 유형

### 1. 일반 업데이트 (standard)

일상적인 콘텐츠 업데이트용

### 2. 긴급 수정 (urgent)

긴급하게 처리된 요청 통보용
- 긴급성 강조
- 빠른 확인 요청

### 3. 대량 업데이트 (bulk)

여러 항목이 한꺼번에 업데이트된 경우
- 요약 테이블 포함
- 상세 목록 별도 섹션

### 4. 기능 추가 (feature)

새로운 기능이 추가된 경우
- 사용 방법 안내 포함
- 스크린샷 첨부 권장

### 5. 버그 수정 (bugfix)

오류/문제 수정 통보용
- 이전 문제 설명
- 수정 후 상태 설명

## 사용법

### 기본 사용

```bash
/deploy-notify
```

워크플로우 컨텍스트에서 자동으로 정보를 수집하여 메일 초안 생성

### 템플릿 지정

```bash
/deploy-notify --template urgent
```

긴급 수정 템플릿 사용

### 미리보기

```bash
/deploy-notify --preview
```

메일 초안만 표시 (저장 안 함)

## 메일 작성 가이드라인

### 톤앤매너

- **공손하고 전문적인 어조** 유지
- 지나치게 격식을 차리지 않되 예의 바르게
- 명확하고 간결한 문장 사용

### 피해야 할 표현

```
❌ "작업 완료함" → ✅ "작업이 완료되었습니다"
❌ "확인 바람" → ✅ "확인 부탁드립니다"
❌ "문제 있으면 연락" → ✅ "문의사항이 있으시면 연락 주세요"
```

### 필수 포함 정보

1. 무엇이 변경되었는지
2. 어디서 확인할 수 있는지
3. 문제 시 연락 방법

### 선택 포함 정보

- 배포 일시
- 관련 티켓 번호
- 첨부 파일 안내

## 커스터마이징

### 발신자 정보 설정

`.workflow/config.json`:
```json
{
  "notify": {
    "sender_name": "웹사이트 관리팀",
    "sender_contact": "contact@example.com"
  }
}
```

### 수신자 정보 설정

```json
{
  "notify": {
    "default_recipient": "입학처 담당자님"
  }
}
```

## 주의사항

1. **자동 발송 안 됨**: 생성된 메일은 반드시 수동으로 발송해야 합니다
2. **민감 정보 주의**: 메일에 내부 기술 정보가 과도하게 포함되지 않도록 주의
3. **첨부 파일**: 스크린샷 등 첨부가 필요한 경우 별도로 준비
4. **수신자 확인**: 발송 전 수신자 주소 반드시 확인
